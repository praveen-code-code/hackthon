#01

Smart Todo Manager

Manage daily tasks with priority levels, due dates, and a productivity score to track completed work.

JWT AuthCRUDPriorityDue DatesProductivity Score
Features
  • User Signup & Login with JWT Authentication
  • Create, Update & Delete Todos
  • Mark tasks as Completed
  • Priority Levels — High / Medium / Low
  • Due Date per task
  • Productivity Score endpoint
Database Tables
users_db
user_id (PK) · username · email UNIQUE · password
todos
todo_id (PK) · user_id (FK) · title · description · priority · status DEFAULT 'Pending' · due_date · created_at
API Endpoints — click any to expand
POST /signup Public ▾
Register a new user account. Password is hashed with bcrypt before storing.
Request Body (JSON)
"username": "john_doe", "email": "john@example.com", "password": "SecurePass123"
Success Response — 201
"message": "User registered successfully", "user_id": 1
Error Responses
// 400 — missing fields "error": "All fields are required" // 409 — email already exists "error": "Email already registered"
POST /login Public ▾
Authenticate user and return a JWT token valid for 24 hours.
Request Body (JSON)
"email": "john@example.com", "password": "SecurePass123"
Success Response — 200
"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user_id": 1, "username":"john_doe"
Error Responses
// 401 — wrong credentials "error": "Invalid email or password"
Notes
  • Send token as Authorization: Bearer <token> in all protected routes
POST /create-todo 🔒 JWT ▾
Create a new todo for the logged-in user. Status defaults to "Pending".
Request Body (JSON)
"title": "Buy groceries", "description": "Milk, eggs, bread", "priority": "High", // High | Medium | Low "due_date": "2026-06-01"
Success Response — 201
"message": "Todo created", "todo_id": 5
Error Responses
// 400 — title missing "error": "Title is required" // 401 — no/invalid token "error": "Unauthorized"
GET /get-todos 🔒 JWT ▾
Fetch all todos belonging to the logged-in user, sorted by created_at descending.
Query Params (optional)
?priority=High // filter by priority ?status=Pending // filter by status
Success Response — 200
"todos": [ { "todo_id": 5, "title": "Buy groceries", "description": "Milk, eggs, bread", "priority": "High", "status": "Pending", "due_date": "2026-06-01", "created_at": "2026-05-13T10:00:00" } ]
PUT /update-todo/<id> 🔒 JWT ▾
Update title, description, priority, or due_date of a specific todo. Only the owner can update.
URL Param
id — todo_id of the record to update
Request Body (JSON) — all fields optional
"title": "Buy groceries & veggies", "priority": "Medium", "due_date": "2026-06-05"
Success Response — 200
"message": "Todo updated successfully"
Error Responses
// 404 — todo not found "error": "Todo not found" // 403 — not the owner "error": "Access denied"
DELETE /delete-todo/<id> 🔒 JWT ▾
Permanently delete a todo by its ID. Only the owner can delete.
URL Param
id — todo_id to delete
Success Response — 200
"message": "Todo deleted successfully"
Error Responses
// 404 "error": "Todo not found"
PUT /complete-todo/<id> 🔒 JWT ▾
Mark a todo as "Completed". Toggles status — if already Completed, sets back to Pending.
URL Param
id — todo_id to mark complete
Success Response — 200
"message": "Todo marked as Completed", "status": "Completed"
GET /productivity-score 🔒 JWT ▾
Returns a productivity score: percentage of todos marked Completed out of total todos created by the user.
Success Response — 200
"total_todos": 20, "completed": 14, "pending": 6, "score_percent": 70.0
Notes
  • Score = (completed / total) × 100, rounded to 1 decimal place

Rules & Guidelines
Read carefully before you start building

This hackathon is organised by Solete. The goal is to build a real, working backend API with a connected frontend for one of the 10 project tracks above. You are free to use any AI tool — ChatGPT, Claude, Gemini, Copilot, or anything else — to help write, debug, and plan your code. What matters is that you understand what you've built.

1️⃣
Pick One Project
Choose any one track from the 10 options. Each participant or team works on a single project. No duplication within a team.
2️⃣
Build the Backend API
Use Flask or FastAPI with PostgreSQL. Implement all listed endpoints with JWT authentication and proper error handling.
3️⃣
Design a Frontend
Create a working UI (HTML/CSS/JS or React) that connects to your API and demonstrates all core features.
4️⃣
Follow the DB Schema
Use the exact table and column names provided. You may add extra columns but all required fields must be present.
5️⃣
AI Tools Are Allowed
ChatGPT, Claude, Gemini — all fine. But during evaluation you must explain your code. If you can't, marks will be deducted.
